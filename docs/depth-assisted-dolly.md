# 深度辅助后退实验

## 分支目标

分支：`feature/depth-assisted-dolly`

本分支用于开发 `docs/suggestion.md` 中的方案五：深度辅助的真实前进 / 后退。

当前主线的前进 / 后退基于 FOV 缩放，能模拟视角变化，但不是相机真实位移。该实验分支的目标是先搭建一条真实相机位移渲染链路：开启深度后退后，前进 / 后退不再扩大或缩小 FOV，而是让相机沿当前视线方向移动。

## 当前实现

### 深度后退开关

工具栏新增 `深度后退` 开关。

开启后：

- 当前镜头焦段的基础 FOV 保持不变。
- `pose.dolly` 会转换为相机沿视线方向的真实位移。
- 前进 / 后退控制仍复用原来的 `dolly` 状态。
- 渲染对象从普通球面全景切换为带深度代理的全景网格。
- `深度强度` 滑条会同时控制网格深度位移幅度和相机移动幅度。

关闭后：

- 回到主线逻辑：通过有效 FOV 模拟前进 / 后退。

如果没有加载深度图：

- `深度后退` 开关保持禁用。
- 预览继续使用主线 FOV 缩放逻辑。
- 不再使用程序化假深度，避免出现来源不明、难以判断的视差。

### 深度图入口

工具栏新增 `选择深度图` 图标按钮。

如果上传深度图：

- 深度图按 equirectangular UV 对齐读取。
- 灰度值会转换为全景网格的半径变化。
- 相机移动时可以产生比普通全景球更明显的视差。
- 上传后会自动打开 `深度后退`，便于直接对比前进 / 后退效果。

## 已知限制

1. 当前没有接入真实深度估计模型。
2. 上传深度图需要用户自己保证与全景图对齐。
3. 相机移动后，原图没有拍到的遮挡区域仍然无法真实补全。
4. 如果深度图质量差，视差可能比 FOV 缩放更不自然。

## 后续方向

建议下一步按以下顺序推进：

1. 增加深度图预览，让用户确认深度图方向、亮暗含义和对齐关系。
2. 增加深度图反相开关，适配不同深度图约定。
3. 研究浏览器端或服务端深度估计模型，生成与全景图对齐的深度图。
4. 在相机移动后增加边缘 / 空洞处理策略。
5. 将实验模式和主线 FOV 缩放模式清晰区分，避免用户误以为它们是同一种“后退”。
