# 全景视角还原与去畸变建议

## 当前问题判断

当前工具的“前进 / 后退”并不是真实改变相机位置，而是通过改变有效 FOV 来模拟镜头前后效果。这个做法可以让画面看起来变近或变远，但它本质上是在同一个全景球中心点上切换视角宽窄。

因此，当用户在 `50mm` 下“后退”并希望看到更多内容时，实际发生的是：`50mm` 的窄视角被逐步变成更广的视角。视角越广，透视投影的边缘拉伸越明显。这个拉伸不是普通意义上的镜头畸变，而是广角透视和球面全景投影带来的必然结果。

换句话说：只靠一张 equirectangular 全景图，无法同时满足“像 50mm 正常镜头一样自然”和“后退看到更多内容”。因为真实后退需要新的相机位置、视差和遮挡关系，而这些信息在单张全景图里不存在。

## 目标重新定义

如果目标是“得到更贴近正常镜头下某些角度的照片”，建议把功能目标从“去畸变”改成更明确的两类：

1. 正常镜头取景：保持接近真实相机的 rectilinear 透视，接受视野较窄，适合导出自然照片。
2. 宽视角展示：允许看到更多内容，但使用更适合全景展开的投影方式，尽量降低边缘拉伸，适合预览和展示空间关系。

这两类目标不应该完全由同一个“去畸变”开关承担。

## 方案一：正常镜头照片模式

做法：新增一个“照片模式”或“标准镜头模式”，在该模式下严格使用焦段对应的 rectilinear FOV，不允许通过“后退”无限增大视野。

建议规则：

- `50mm / 85mm / 100mm / 135mm` 保持窄视角，只允许小幅微调。
- 当用户需要看到更多内容时，明确提示切换到 `35mm / 25mm`，而不是让 `50mm` 继续后退。
- 将当前“前进 / 后退”在照片模式中改名为“放大 / 缩小”或“视角微调”，避免暗示真实相机位移。
- 导出时保留 16:9，但本质上是从全景球中裁出一个正常透视窗口。

优点：

- 最接近真实相机照片。
- 行为最容易解释。
- 实现成本低，当前架构基本支持。

缺点：

- 无法在 `50mm` 下看到很多内容。
- 用户想要“既自然又广”的需求无法完全满足。

建议优先级：高。它应该成为“正常照片导出”的基础模式。

## 方案二：投影模式选择

做法：把“去畸变”升级成“投影模式”，提供多个投影选项，而不是只用一个强度滑条。

建议投影：

- 标准透视：当前正常相机视图，适合照片感最强的导出。
- 柱面投影：水平边缘拉伸更少，适合横向看更多空间，但垂直方向仍需克制。
- Panini / Vedutismo 近似：建筑和室内常用的宽视角折中投影，能在保留中心透视感的同时缓解边缘拉伸。
- 全景预览：作为导航和找角度用，不作为正常照片导出。

优点：

- 能把“照片感”和“看更多”拆成可选择的产品语义。
- 用户可以根据场景选择，而不是反复调一个难以理解的畸变滑条。
- 比当前单一去畸变更符合实际摄影/全景工具的工作流。

缺点：

- 每种投影都有边界，没有一种能在大视角下完全像正常 50mm。
- 需要为导出和预览统一投影实现，避免所见非所得。

建议优先级：高。建议作为下一轮主要改造方向。

## 方案三：多张窄视角拼接导出

做法：用户仍选择一个正常焦段，例如 `50mm`，但工具不是把 FOV 强行拉宽，而是从多个相邻 yaw / pitch 角度分别渲染窄视角照片，再拼成一张更宽的图。

示例：

- 选择 `50mm` 中心角度。
- 工具自动渲染左、中、右三张 50mm 视角。
- 拼成一张更宽的 16:9 或 21:9 图。

这类似用正常镜头拍多张照片后拼接。局部物体会比单张超广角自然，但整张图严格来说不再是单一相机的一次曝光。

优点：

- 比直接增大 FOV 更接近“正常镜头看更多”的直觉。
- 很适合室内横向空间展示。
- 不需要深度图，复杂度中等。

缺点：

- 拼接处可能有透视不连续。
- 画面不是严格物理相机模型。
- 对边缘内容和近景物体仍可能有不自然感。

建议优先级：中高。可以作为“宽幅照片导出”功能探索。

## 方案四：限制风险区域并给用户反馈

做法：不要试图在所有参数下都强行生成“正常照片”。当有效 FOV 太大时，在 UI 上给出风险提示。

建议规则：

- 有效 FOV 小于 `50`：标记为“照片感较好”。
- 有效 FOV 在 `50-75`：标记为“轻微广角拉伸”。
- 有效 FOV 大于 `75`：标记为“宽视角，边缘会明显变形”。

可以在状态栏显示：

`FOV 68 deg / 轻微广角`

优点：

- 帮助用户理解为什么画面变形。
- 避免把不可避免的投影问题伪装成 bug。
- 实现成本低。

缺点：

- 只解释问题，不解决问题。

建议优先级：中。适合和方案一、二一起做。

## 方案五：深度辅助的真实后退

做法：为全景图估计深度，把全景图投到一个近似 3D 网格或点云上，再让相机做真实位置移动。

可能路线：

- 使用深度估计模型生成全景深度图。
- 将全景图转为带深度的球面 / mesh。
- 前进后退时移动相机，产生有限视差。
- 空洞区域用插值或生成式补全。

优点：

- 这是最接近“真实后退”的方向。
- 如果深度质量好，可以显著改善单纯 FOV 缩放的问题。

缺点：

- 实现复杂度高。
- 浏览器端性能和模型体积是问题。
- 深度估计在室内细节、遮挡、桌椅边缘上容易出错。
- 后退后原图没有拍到的遮挡区域必须补洞，可能出现明显瑕疵。

建议优先级：低到中。适合作为实验分支，不建议直接作为主功能承诺。

## 方案六：AI 扩图 / 生成式补全

做法：保持 `50mm` 正常镜头视角作为中心图，不通过增大 FOV 看更多，而是在导出阶段对画面四周做生成式扩展。

优点：

- 视觉上最容易得到“正常镜头但画面更宽”的结果。
- 对海报、概念图、非严肃测量用途可能很有用。

缺点：

- 生成内容不再是真实全景图里的内容。
- 不适合要求忠实还原的场景。
- 需要引入生成式模型或外部服务。

建议优先级：低。除非产品定位允许“创作增强”，否则不建议优先做。

## 推荐路线

建议下一阶段不要继续强化当前“去畸变”滑条，而是调整产品结构：

1. 保留当前滑条作为高级微调工具，但不再把它当作主要解决方案。
2. 新增“照片模式”：按焦段输出标准透视视图，限制过大的后退缩放。
3. 新增“投影模式”：标准透视、柱面、Panini 三选一。
4. 在状态栏显示当前有效 FOV 和变形风险。
5. 后续再尝试“多张窄视角拼接导出”，作为解决“正常镜头看更多”的折中方案。

如果只选一个最务实的改造方向，建议优先做：

`照片模式 + 投影模式选择 + FOV 风险提示`

原因是它能把当前混在一起的问题拆开：正常照片就接受窄视角，宽视角展示就选择更合适的投影，不再用“去畸变”这个单一概念承担所有预期。
